mac连接ssh常见错误

约 420 字大约 1 分钟

mac

2023-11-24

错误内容： no matching host key type found. Their offer: ssh-rsa,ssh-dss

  1. 在 Mac 上处理此错误消息的方法与在其他操作系统上处理类似。您可以尝试以下两种解决方案之一：

升级您的 SSH 客户端：一些旧版本的 SSH 客户端可能不支持某些主机密钥类型。您可以尝试升级您的 SSH 客户端到最新版本，以查看它是否支持远程服务器提供的主机密钥类型。

  1. 启用主机密钥类型：您还可以在 SSH 客户端配置中启用远程服务器提供的特定主机密钥类型。例如，如果远程服务器提供 ssh-rsa 和 ssh-dss，则可以将以下行添加到您的 SSH 配置文件中：
Host servername
HostKeyAlgorithms +ssh-rsa,ssh-dss

将 servername 替换为远程服务器的实际主机名或 IP 地址。这将为连接到此服务器的连接启用指定的主机密钥算法。请注意，启用弱密码或已废弃的主机密钥算法可能会危及 SSH 连接的安全性。

在 Mac 上，您可以通过终端应用程序访问 SSH 客户端配置文件 (~/.ssh/config) 并进行编辑。 若要打开该文件，请在终端中键入以下命令并按 Enter 键：

nano ~/.ssh/config

这将使用 Nano 编辑器打开 SSH 客户端配置文件。您可以在文件末尾添加上面的代码段。完成后，按 Control + X 键，然后按 Y 键保存并退出编辑器。
