Linux 常用命令

约 458 字大约 2 分钟

linux

2023-08-03

1. 查看端口号使用情况：

1. lsof -i:端口号
2. netstat -tunlp|grep 端口号

2. 杀死进程

kill -9 Pid

3. 查找进程

ps -ef | grep java

4. 查看日志

# 动态查看日志
tail -f info.log
# cat 是由第一行到最后一行连续显示在屏幕上
cat info.log

5. 日志文件搜索

# 查找日志文件中关键词部分 -A 50 上下内容指定行数
grep -A 50 'ee7bf2d27c1c4e44' wlhy.log

# 查找日志文件中关键词部分
cat -n app-weihuapin.20230403_0.log |grep "报文解析" 
cat -n app-weihuapin.20230403_0.log |grep "设备表中未查询到"

# 将匹配的内容输出到本目录到bug.log (bug.log不存在新建，存在覆盖)
cat -n app-weihuapin.20230403_0.log |grep "33908b116df60f92" >./bug.log

6.切换用户

su - root 切换root用户

7.全局查找文件

在Linux系统中，可以通过终端运行以下命令来查找Nginx配置文件的位置：

find / -name "nginx.conf"
# 指定目录查找文件
find /path/to/search -name file.txt

这个命令将在整个文件系统中查找名为 nginx.conf 的文件。请注意，由于该命令需要搜索整个文件系统，所以可能需要一些时间来完成。

如果找到了 Nginx 配置文件，请记得备份它们并小心修改它们，以免导致您的 Nginx 服务器无法正常工作。

8.文件内容替换

 sed -i "" -e 's/SHCZ_YSGL_FISCAL/SHCZ_YSGL_FISCAL_MH/g' *.sql
 sed 's/被替换内容/替换成/g' 原始文件 > 生成文件
 sed 's/java/-/g' a.txt > b.txt

9.文件复制

# 复制目录下的所有文件到 指定目录下（注意覆盖问题）
cd ~/docker/zey.51shanjian/zey-dev/repository/doc/docker/nfs
sudo cp -r assets/* ~/docker/zey.51shanjian/zey-dev/nfs/assets

# 复制文件到 指定目录下（注意覆盖问题）
sudo cp  apple-app-site-association ~/docker/zey.51shanjian/zey-sit/nfs/apple-app-site-association
